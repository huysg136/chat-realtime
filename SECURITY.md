# Security Policy

## Supported Versions

As **Quik** is currently in the **Work In Progress (WIP)** stage, we prioritize security updates for the latest development branch.

| Version | Supported          |
| ------- | ------------------ |
| Latest (main) | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Quik seriously. If you discover a security vulnerability, please help us fix it by following these steps:

1. **Do Not Open a Public Issue:** To prevent exploitation of the vulnerability, please do not report security bugs through GitHub Issues.
2. **Contact Us:** Send a detailed report to our team at **support@quik.id.vn**.
3. **What to Include:**
   - A description of the vulnerability.
   - Step-by-step instructions to reproduce the issue.
   - Any potential impact on users or data (e.g., Firebase rules, Stringee tokens, or Cloudflare R2 access).

### What to Expect
- **Acknowledgment:** You will receive an automated confirmation of your report within **24-48 hours**.
- **Assessment:** Our team will investigate and confirm the vulnerability.
- **Updates:** We will provide regular updates on the status of the fix.
- **Public Disclosure:** Once the fix is deployed, we may publicly acknowledge your contribution (if desired) in our release notes.

Thank you for helping keep Quik secure!
